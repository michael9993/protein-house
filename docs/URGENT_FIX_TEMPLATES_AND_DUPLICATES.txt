ISSUE #1: Templates Are Stored in Dashboard, Not Code!
───────────────────────────────────────────────────────────────────────────

❌ PROBLEM:
   The email templates are NOT loaded from the code file!
   They're stored in the SALEOR DASHBOARD CONFIGURATION (database).
   
   The default-templates.ts file is only used when creating a 
   NEW configuration.
   
   Your existing SMTP configuration still has the OLD templates!

✅ SOLUTION:
   Delete and recreate your SMTP configuration in the Dashboard.
   The new config will automatically use the new professional templates!

═══════════════════════════════════════════════════════════════════════════

ISSUE #2: Duplicate Emails Being Sent
───────────────────────────────────────────────────────────────────────────

❌ PROBLEM:
   You're receiving multiple copies of the same email.
   This is caused by either:
   - Multiple SMTP configurations (all active)
   - Duplicate webhook registrations for the same event

✅ SOLUTION:
   Keep only ONE active SMTP configuration.
   Delete any duplicate webhook registrations.

STEP 1: Save Your Current SMTP Settings
───────────────────────────────────────────────────────────────────────────

Before deleting anything, write these down:

□ SMTP Host: _______________________
□ SMTP Port: _______________________
□ SMTP Username: ___________________
□ SMTP Password: ___________________

STEP 2: Delete Old SMTP Configuration
───────────────────────────────────────────────────────────────────────────

1. Go to: Dashboard → Apps → SMTP
2. Find your existing configuration
3. Click DELETE or trash icon
4. Confirm deletion

───────────────────────────────────────────────────────────────────────────

STEP 3: Create New SMTP Configuration
───────────────────────────────────────────────────────────────────────────

1. Click "Add Configuration" or "Create New"
2. Enter the settings you wrote down in Step 1
3. Name it (e.g., "Production Email")
4. Configure SMTP server details
5. Select channels
6. DO NOT customize templates - they'll use the new ones!
7. Save

───────────────────────────────────────────────────────────────────────────

STEP 4: Check for Duplicate Configurations
───────────────────────────────────────────────────────────────────────────

1. In Dashboard → Apps → SMTP
2. Make sure you only have ONE configuration
3. If you see more than one:
   - Delete the extras
   - Keep only ONE active

───────────────────────────────────────────────────────────────────────────

STEP 5: Check for Duplicate Webhooks (Optional but Recommended)
───────────────────────────────────────────────────────────────────────────

Use the GraphQL query in:
   infra/check-smtp-duplicates.graphql

This will show you if there are duplicate webhook registrations.

───────────────────────────────────────────────────────────────────────────

STEP 6: Test!
───────────────────────────────────────────────────────────────────────────

💡 WHY THIS HAPPENED:

═══════════════════════════════════════════════════════════════════════════

The SMTP app stores email templates in the CONFIGURATION, not just
in the code. This is by design - it allows you to customize templates
per configuration in the Dashboard UI.

When I updated default-templates.ts, it only affects NEW configurations
that you create. Your existing configuration still had the old templates
stored in the database.

The proper workflow is:
1. Update default-templates.ts (✅ Done)
2. Delete old configuration (⏳ You need to do this)
3. Create new configuration (⏳ You need to do this)
4. New config uses updated templates automatically (✅ Will happen)
